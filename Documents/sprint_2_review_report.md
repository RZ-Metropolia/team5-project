# Sprint 2 Review Report

## Sprint Goal
1. Implement the database and verify CRUD operations
2. Start developing the user interface
3. Integrate unit testing
4. Configure code coverage testing

## Completed Tasks
1. Designed and created the relational database schema using MariaDB.
2. Defined tables with fields and relationships, and seeded the database with sample data.
3. Verified that Create, Read, Update, and Delete (CRUD) operations function correctly.
4. Began frontend development, implementing initial views and focusing on layout and basic interactivity.
5. Wrote unit tests for key functions following best practices, and included them in the repository.
6. Configured code coverage testing using JaCoCo, ensuring coverage of critical functionalities.
7. Exported the HTML code coverage report and published it to a public folder.

## Demo Summary
The following things are presented during the sprint 2 review:
1. The database schema and the generation script.
2. A demonstration of the CRUD operations working with the seeded sample data.
3. The initial frontend screens and interactive layout elements.
4. The execution and results of the implemented unit tests.
5. The JaCoCo HTML code coverage report.

## What Went Well
1. The database implementation and setup went smoothly, providing a solid foundation.
2. The initial user interface closely aligns with the product vision.
3. The integration of unit testing and JaCoCo coverage reporting establishes a strong quality baseline early in the project.

## What could be improved
1. Continue to expand unit tests to cover more complex logic and edge cases.
2. Ensure frontend interactivity keeps pace with the backend data models.
